**A09:2025- SECURITY LOGGING AND ALERTING FAILURES**

<img width="1040" height="429" alt="Security-Logging-and-monitoring-failures-attack-example" src="https://github.com/user-attachments/assets/8cb7a463-6008-4377-9853-18c4299b18b0" />


**Descripción:** Esta vulnerabilidad no se trata de un error de código que "rompe" algo directamente, sino de la incapacidad de la aplicación para detectar y responder a un ataque en curso. Ocurre cuando los eventos críticos de seguridad (logins fallidos, ataques de fuerza bruta, cambios de privilegios) no se registran, se registran de forma insuficiente o las alertas no llegan a tiempo a los responsables.

**Causas:**

-   Eventos auditables (como inicios de sesión o transacciones de alto valor) no se guardan en logs.
    
-   Los logs solo se almacenan localmente y el atacante los borra.
    
-   Las alertas son tan frecuentes que el personal de seguridad las ignora.
    
-   No hay monitoreo en tiempo real.
    

**Impacto Potencial:** Sin registros ni alertas, un atacante puede permanecer meses dentro de una red sin ser detectado (lo que se conoce como dwel time o tiempo de permanencia). Esto permite la extracción masiva de datos y el movimiento lateral a otros servidores.

**Métodos de Explotación:** Los atacantes no "explotan" el log, sino que se aprovechan de su ausencia para:

1.  **Fuerza Bruta Indetectable:** Probar miles de contraseñas sin que nadie reciba una alerta de "múltiples intentos fallidos".
    
2.  **Manipulación de Logs (Log Injection):** Si la aplicación registra datos de entrada del usuario sin limpiar, un atacante puede insertar entradas falsas en el log para confundir a los administradores o inyectar scripts maliciosos.
    
![yahoo](https://github.com/user-attachments/assets/11a48e16-49c0-41f8-948d-60c9ef3fbb49)


**Caso Real: Yahoo (2013-2014)**

-   **El incidente:** Yahoo sufrió el robo de datos de 3 mil millones de cuentas.
    
-   **La falla:** Aunque tenían sistemas de seguridad, el registro y monitoreo fallaron al detectar que los atacantes habían creado "cookies falsas" para acceder a las cuentas sin contraseña. Los atacantes estuvieron dentro de los sistemas por años antes de ser descubiertos.
     

**Prevención:**

-   Asegurar que todos los fallos de control de acceso y validación de entrada se registren.
    
-   Utilizar soluciones SIEM(Security Information and Event Management) para centralizar logs.
    
-   Establecer umbrales de alerta efectivos para detectar ataques de fuerza bruta o escaneos.
